# 日志详情页面禁用修复

## 问题描述

用户点击日志列表中的ID链接想查看详情时报错：
```
AttributeError: 'FakeQuerySet' object has no attribute 'get'
```

**原因**：
1. FakeQuerySet缺少`get()`方法
2. 日志应该是只读的，不应该有详情页面

## 修复方案

禁用日志详情页面访问，只允许在列表中查看日志。

---

## 修复内容

### 1. 添加has_view_permission方法（apps/system/admin.py:633-645）

**作用**：控制查看权限
- 允许列表查看（obj=None）
- 禁止单个对象详情查看（obj不为None）
- 自动移除ID列的点击链接

**代码**：
```python
def has_view_permission(self, request: HttpRequest, obj=None) -> bool:
    """
    禁止查看单个日志详情（只允许列表查看）

    返回False会：
    1. 移除ID列的点击链接
    2. 禁止访问详情页面
    3. 用户只能在列表页查看日志
    """
    # 允许列表查看，但禁止单个对象详情查看
    if obj is not None:
        return False
    return request.user.is_authenticated and request.user.is_staff
```

### 2. 添加FakeQuerySet.get()方法（apps/system/admin.py:859-867）

**作用**：作为兜底，如果有代码尝试调用get()，会抛出清晰的异常

**代码**：
```python
def get(self, **kwargs):
    """
    模拟QuerySet.get()方法

    由于日志是只读的，不应该查看单个对象详情，
    这个方法会抛出DoesNotExist异常
    """
    from django.core.exceptions import ObjectDoesNotExist
    raise ObjectDoesNotExist("日志对象不支持单独查看，请在列表中查看")
```

## 技术细节

### Django Admin权限系统

Django Admin通过以下方法控制访问权限：

| 方法 | 作用 | 返回True时 | 返回False时 |
|------|------|-----------|------------|
| `has_add_permission` | 控制添加权限 | 显示"添加"按钮 | 隐藏"添加"按钮 |
| `has_change_permission` | 控制修改权限 | 可以修改对象 | 不能修改对象 |
| `has_delete_permission` | 控制删除权限 | 显示删除按钮 | 隐藏删除按钮 |
| `has_view_permission` | 控制查看权限 | **ID显示为链接** | **ID显示为文本** |

**关键点**：`has_view_permission(request, obj)`的`obj`参数：
- `obj=None`：检查列表查看权限
- `obj=具体对象`：检查单个对象详情查看权限

### FakeQuerySet.get()方法

虽然`has_view_permission`已经阻止了访问，但作为防御性编程，还是添加了`get()`方法：

```python
def get(self, **kwargs):
    from django.core.exceptions import ObjectDoesNotExist
    raise ObjectDoesNotExist("日志对象不支持单独查看，请在列表中查看")
```

这样即使有代码绕过权限检查尝试调用`get()`，也会得到清晰的错误提示。

## 相关方法总结

SystemLogAdmin现在有以下权限控制方法：

```python
class SystemLogAdmin(admin.ModelAdmin):
    # 禁用添加
    def has_add_permission(self, request):
        return False

    # 禁用修改
    def has_change_permission(self, request, obj=None):
        return False

    # 禁用删除
    def has_delete_permission(self, request, obj=None):
        return False

    # 禁用单个对象查看（允许列表查看）
    def has_view_permission(self, request, obj=None):
        if obj is not None:  # 单个对象
            return False
        return request.user.is_authenticated and request.user.is_staff
```

## 常见问题

### Q1: 为什么不让用户查看日志详情？

**A:** 有以下几个原因：
1. **信息完整性**：列表页已经显示了所有9个字段，详情页不会有额外信息
2. **技术简化**：避免为非数据库对象实现完整的CRUD逻辑
3. **安全性**：减少攻击面，防止潜在的信息泄露
4. **用户体验**：列表页直接查看更高效，不需要来回跳转

### Q2: 如果真的需要查看某条日志的详细信息怎么办？

**A:** 可以使用以下方式：
1. **列表页搜索**：使用搜索框按消息/用户名/IP搜索
2. **使用过滤器**：按级别/类型/来源/账户类型过滤
3. **导出功能**：导出为TXT/CSV/JSON查看完整信息
4. **直接查看日志文件**：`logs/system.log`

### Q3: 能否单独为某些管理员开放详情查看权限？

**A:** 可以，修改`has_view_permission()`方法：
```python
def has_view_permission(self, request, obj=None):
    # 超级管理员可以查看详情
    if request.user.is_superuser:
        return True

    # 其他管理员只能查看列表
    if obj is not None:
        return False
    return request.user.is_staff
```

但需要注意：开放详情权限后，需要实现`FakeQuerySet.get()`的完整逻辑，而不是抛出异常。
