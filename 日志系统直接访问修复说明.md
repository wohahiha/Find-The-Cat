# 日志系统直接访问修复说明

## 问题描述

原先的日志系统使用三级分类结构（SystemLogCategoryAdmin），需要通过以下步骤访问：
1. 访问 `/admin/configs/systemlogcategory/`
2. 点击"系统运行日志"
3. 才能看到日志列表

用户要求：**直接访问日志列表，移除三级分类页面**

### 3. 添加_meta属性到LogEntry（apps/system/log_models.py:75-77）

LogEntry是普通Python类，需要模拟Django模型的行为才能在Admin中显示。

**添加代码：**
```python
# 添加_meta属性（从SystemLog模型获取）
from apps.system.models import SystemLog
self._meta = SystemLog._meta
```

### 4. 添加serializable_value方法（apps/system/log_models.py:79-89）

Django Admin需要这个方法来序列化字段值。

**添加代码：**
```python
def serializable_value(self, field_name: str):
    """
    返回字段的可序列化值（Django Admin需要）

    Args:
        field_name: 字段名

    Returns:
        字段值
    """
    return getattr(self, field_name, None)
```

### 5. 为所有字段添加display方法（apps/system/admin.py:633-676）

由于LogEntry不是真正的Django模型，list_display中的字段必须是callable方法。

**添加了9个display方法：**
```python
@admin.display(description="ID")
def id(self, obj) -> int:
    """返回日志ID"""
    return obj.id

@admin.display(description="时间戳")
def timestamp(self, obj) -> str:
    """返回时间戳"""
    return obj.timestamp

@admin.display(description="级别")
def level(self, obj) -> str:
    """返回日志级别"""
    return obj.level

@admin.display(description="类型")
def log_type(self, obj) -> str:
    """返回日志类型：业务日志/框架日志/系统日志/其他"""
    return obj.get_type()

@admin.display(description="来源")
def log_source(self, obj) -> str:
    """返回日志来源（模块名）"""
    return obj.get_source()

@admin.display(description="消息摘要")
def message_summary(self, obj) -> str:
    """返回消息摘要（智能截断）"""
    return obj.get_summary(max_length=100)

@admin.display(description="账户名")
def username(self, obj) -> str:
    """返回账户名"""
    return obj.username or "-"

@admin.display(description="账户ID")
def account_id(self, obj) -> str:
    """返回账户ID"""
    return str(obj.account_id) if obj.account_id is not None else "-"

@admin.display(description="IP地址")
def ip_address(self, obj) -> str:
    """返回IP地址"""
    return obj.ip_address or "-"
```

## 技术细节

### LogEntry vs Django Model

LogEntry不是真正的Django Model，而是一个普通Python类，但为了在Admin中使用，需要模拟以下行为：

| 属性/方法 | 作用 | 实现方式 |
|---------|------|---------|
| `_meta` | 模型元数据 | 从SystemLog模型获取 |
| `_state` | 模型状态 | 创建fake对象：`{'adding': False, 'db': 'default'}` |
| `pk` | 主键 | 等于`id`字段 |
| `serializable_value()` | 序列化字段值 | 使用`getattr()`动态获取属性 |

### FakeQuerySet

为了让非数据库数据能在Admin中显示，实现了`FakeQuerySet`类：

```python
class FakeQuerySet(list):
    """伪造的 QuerySet，用于在 Admin 中显示非数据库数据"""
    def __init__(self, data):
        super().__init__(data)
        self.model = SystemLog
        self.ordered = True
        self._result_cache = data
        self.query = FakeQuery()

    # 实现Django QuerySet的所有必要方法
    def count(self): ...
    def order_by(self, *args, **kwargs): ...
    def filter(self, *args, **kwargs): ...
    # ... 等等
```

### 数据流

```
日志文件(system.log)
    ↓
LogParser.parse_file()
    ↓
LogEntry对象（带_meta、_state、serializable_value）
    ↓
FakeQuerySet（模拟Django QuerySet）
    ↓
SystemLogAdmin.get_queryset()
    ↓
过滤器处理（级别/类型/来源/账户类型）
    ↓
Django Admin渲染（9个display方法）
    ↓
用户界面
```
